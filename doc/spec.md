#方針

 - GLSLとHLSLの共通部分集合を基本に作成する。
 - HLSLのオブジェクト指向的な書き方はGLSLで再現が難しいので省く
 
#訳語
 - expression: 式
 - statement: 文
 - syntax: 構文
 - semantics: セマンティクス
 
#変数

##宣言
*(修飾子)* (型名) (変数名) *:(セマンティクス)* *= (初期化式)*;

##型名
###スカラ型
 - bool
 - int
 - float
 - uint
 - double

###ベクトル型
 - (スカラ型名)(2-4)

### 行列型名
 - mat(2-4)

### swizzle演算
 - (変数名)./[xyzw]{1,4}/
 - (変数名)./[rgba]{1,4}/

### 修飾子
 - const
 - static
 - uniform


## 構造体
struct *型名*{ *+(メンバ宣言)* };

 - 型扱い、ただし初期化子リストへの対応は要検討
 - 構造体の入れ子や無名構造体は不可能
 - ターゲット言語由来の不可能な修飾等は個別対応

## 構文
### if
if((条件式)) (文) *else (文)*

### for
for((初期化式));(条件式);(再初期化式)) 文
 - break, continueの解釈はターゲット言語任せ? 結合くらいは見てもいいかも

### while
while((条件式)) 文

### do-while
do (文) while((条件式));

## 属性
[[(属性名) *= (値)* ) *, (属性名)= (値)*]]
 - PSとVSの分離
 - 各main関数(複数回定義ならその回数も)の指定
 - 機能の一部をユーザ定義に以上する場合の関数指定
